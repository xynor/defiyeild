# AGENTS.md

Aave USDC yield tracker — daily on-chain snapshots via Vercel Cron, stored in Supabase, displayed with Recharts. Full details: [README.md](./README.md).

## Commands

```bash
npm run dev      # http://localhost:3000
npm run build    # production build → .next/
npm run start    # serve production build
npm run lint     # ESLint
```

No test command is configured.

## Architecture

```
Vercel Cron (daily 00:00 UTC)
  → POST /api/cron (Bearer CRON_SECRET)
    → ethers.js reads aEthUSDC.balanceOf from Ethereum RPC
    → Prisma upserts into Supabase PostgreSQL (balance_snapshots table)
      → GET /api/snapshots?address=0x...&days=7
        → app/page.tsx (client component) renders YieldChart (Recharts)
```

## Conventions

- **Import alias**: `@/*` maps to project root (`import { YieldChart } from '@/components/YieldChart'`)
- **Naming**: camelCase for variables/functions, PascalCase for components/interfaces
- **Styling**: Tailwind utility-first, dark theme (gray-950 background). No CSS modules.
- **USDC display**: divide on-chain values by `1_000_000` (6 decimals) for user-facing numbers
- **Dates**: ISO strings in DB (`YYYY-MM-DD`), formatted as `"MM-DD"` on the chart
- **Big numbers**: stored as `Decimal(78,0)` in DB (no precision loss). Always convert ethers `bigint` → `string` → Prisma `Decimal`.
- **Earnings**: `dailyEarnings = today.balance - yesterday.balance`, `cumulativeEarnings = today.balance - firstDay.balance`

## Environment Variables

All are **required**. No `.env.local.example` exists; setup is documented in [README.md](./README.md#2-environment-variables).

| Variable                        | Where used                                  |
| ------------------------------- | ------------------------------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`      | Browser + server                            |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Browser + server                            |
| `SUPABASE_SERVICE_KEY`          | Server only (cron writes)                   |
| `NEXT_PUBLIC_WALLET_ADDRESS`    | Frontend (single wallet, hardcoded)         |
| `RPC_URL`                       | Server only (Ethereum Mainnet)              |
| `CRON_SECRET`                   | Server only (Bearer auth on /api/cron)      |
| `DATABASE_URL`                  | Prisma (usually auto-generated by Supabase) |

## Pitfalls
Cron idempotency**: Same-date snapshot overwrites (upsert on `@@unique([address, snapshotDate])`). No automatic backfill if cron fails.
2. **RLS is wide open**: `supabase_migration.sql` allows anonymous read on the entire table. If privacy matters, tighten RLS policies before production.
3. **Single wallet only**: `NEXT_PUBLIC_WALLET_ADDRESS` is one address. Tracking multiple wallets requires code changes in `app/page.tsx` and the cron route.

5. **Chart requires data**: First cron run must complete before the chart renders. The page hardcodes `days=7` for the snapshot query.

## Key Files

| File                         | Role                                                                   |
| ---------------------------- | ---------------------------------------------------------------------- |
| `lib/aave.ts`                | On-chain reads (scaledBalance, liquidityIndex, underlyingValue)        |
| `lib/db.ts`                  | High-level DB (aEthUSDC.balanceOf)                                     |
| `lib/prisma.ts`              | Prisma singleton (prevents Next.js HMR connection leaks)               |
| `lib/supabase.ts`            | Supabase client factory (server + browser variants)                    |
| `app/api/cron/route.ts`      | Daily snapshot writer (validates CRON_SECRET, reads chain, upserts DB) |
| `app/api/snapshots/route.ts` | Frontend data API (query by address + days)                            |
| `app/page.tsx`               | Homepage — summary cards + YieldChart, client component                |
| `components/YieldChart.tsx`  | Dual-axis Recharts AreaChart (balance left, earnings right)            |
| `prisma/schema.prisma`       | Single `BalanceSnapshot` model                                         |
| `supabase_migration.sql`     | Raw SQL migration (run in Supabase SQL Editor)                         |
| `vercel.json`                | Cron schedule definition                                               |
